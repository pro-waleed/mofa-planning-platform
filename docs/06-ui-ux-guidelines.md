# 06 UI UX Guidelines

## Experience Direction

The interface is designed as an executive institutional workspace rather than a generic admin panel. The experience should feel calm, formal, readable, and presentation-ready.

## Core UX Principles

- Arabic-first by default
- genuine RTL layout, not mirrored as an afterthought
- spacious composition with strong visual hierarchy
- executive summaries before operational detail
- consistency in labels, status language, and actions
- minimum clutter and no decorative dashboard noise

## Layout Guidelines

- prioritize desktop and meeting-room presentation scenarios
- keep content width controlled for readability
- use a persistent application shell and clear navigation zones
- group related actions inside cards or panels
- avoid crowded tables when cards or sections communicate better in Arabic

## Typography And Language

- use formal, concise Arabic labels
- avoid mixing English into primary UI copy unless technical
- keep action verbs direct and administrative
- prefer clear status labels such as `مسودة`, `معتمد`, `معاد`, `متعثر`

## Dashboard Guidelines

- highlight only a few top metrics
- emphasize exceptions, approvals, and delays
- use charts sparingly and with clear purpose
- support quick action entry points for demos and real work

## Form Design Guidelines

- keep labels above fields
- show validation clearly and politely
- separate metadata from workflow actions
- use focused sections for long forms such as mission reports
- preserve strong contrast and readable input height

## Status And Workflow Design

- every approval or review state must be visible
- returned items must show why they were returned
- delayed items should be unmistakable without overwhelming the page
- action buttons should reflect institutional language and consequences

## Empty, Loading, And Validation States

- empty states should explain the next administrative action
- loading states should preserve layout stability
- validation states should point to the exact missing information
- workflow states should make ownership and responsibility obvious

## Accessibility Considerations

- maintain sufficient contrast
- avoid relying on color only for status meaning
- support keyboard navigation for forms
- preserve readable spacing in Arabic text blocks

## Future UX Enhancements

- richer drag-and-drop plan editing
- printable executive summaries
- saved dashboard views by role
- calendar and deadline visualizations
- inline evidence attachment experience
